# 🎮 Mobile Game Player Behavior Analysis

This project analyzes player behavior in a mobile game using event‑level data.  
It focuses on understanding user engagement, retention, churn, and long‑term activity patterns through data exploration and visualization.

---

## 📁 Project Structure

| File/Folder | Description |
|------------|-------------|
| `Player Behavior Analysis.ipynb` | Main analysis notebook |
| `events.csv` | In‑game event logs |
| `users.csv` | User metadata |
| `plots` | All generated visualizations (DAU, WAU, MAU, retention, churn, cohorts) 

---

## 📊 Analysis Performed

### **1️⃣ Daily Active Users (DAU)**
- Measures unique active users per day  
- Helps identify engagement trends and activity spikes  

### **2️⃣ Weekly Active Users (WAU)**
- Aggregates user activity at a weekly level  
- Shows medium‑term engagement patterns  

### **3️⃣ Monthly Active Users (MAU)**
- Long‑term engagement metric  
- Useful for understanding overall game health  

---

### **4️⃣ Retention Analysis (D1, D7, D30)**
- Calculates how many users return after:
  - **Day 1**
  - **Day 7**
  - **Day 30**
- Helps measure early‑stage user stickiness and long‑term retention  

---

### **5️⃣ Churn Analysis**
- Identifies users who stop playing after a certain period  
- Highlights churn patterns and potential risk indicators  

---

### **6️⃣ Cohort Analysis**
- Groups users by **install month**  
- Tracks their activity over time  
- Visualized using a cohort heatmap  
- Helps compare user groups and understand lifecycle behavior

## 📌 Summary

This analysis provides insights into:

- How often players return to the game  
- How retention changes over time  
- Which user cohorts perform better  
- When and why users churn  
- Overall engagement trends (daily, weekly, monthly)

These insights can help game developers improve user experience, retention strategies, and product decisions.
